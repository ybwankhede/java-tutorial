Variables

Definition:
A variable in Java is a named memory location that stores data. It is like a container that holds values which may change during program execution. In Java, every variable has a type that determines the size and layout of the variable's memory.

Simple Understanding

Think of a variable as a labeled jar where you keep things. Example: A jar labeled sugar contains 2 kg sugar. Tomorrow, you can refill it with 3 kg sugar.

Types of variables

  • Local variables — declared inside a method or block. They must be initialized before use and their scope ends when the block/method finishes.
  • Instance (non-static) variables — declared at class level without static. Each object has its own copy. Default values are assigned when an object is created.
  • Static (class) variables — declared with static. Shared by all instances of the class and stored in the class area.
  • Final variables (constants) — declared with final. Value cannot change once assigned. Use for constants.

Syntax & Types

Type Syntax Example
Local Variable datatype varName = value; int age = 25;
Instance Variable Declared inside class but outside methods String name;
Static Variable static datatype varName; static int count = 0;
Final Variable (Constant) final datatype varName = value; final double PI = 3.14159;

Real-Life Example

Imagine a school:

  • Static (class) Variable: "School Name" → Same for all students.
  • Instance (non-static) Variable: "Student Name" → Different for each student.
  • Final variables (constants) — "Enrollment Number" -> The enrollment number is permanent and will not change but Different for each student
  • Local Variable: "Exam Score (temporary)" → Exists only during exam calculation.

Example

  class Student {
      // Instance variable
      String name;
      
      // Final variable
      final String enrollmentNumber;;

      // Static variable
      static String school = "ABC School";

      // Constructor
      Student(String n) {
          name = n; // local variable
      }

      public void display() {
          System.out.println(name + " studies at " + school);
      }

      public static void main(String[] args) {
          Student s1 = new Student("Ravi");
          Student s2 = new Student("Priya");

          s1.display(); // Ravi studies at ABC School
          s2.display(); // Priya studies at ABC School
      }
  }

Advantages

  • Provide data storage inside programs.
  • Helps in memory management (local scope frees quickly).
  • Supports code readability with meaningful names.

Disadvantages

  • Wrong use of static variables may cause memory leaks.
  • Overuse of global/static variables reduces modularity.

Specific Notes

  • All variables must be declared before use.
  • Default values: 0 for numbers, false for boolean, null for objects.
  • Local variables must be initialized before use (no default value).

Latest Java Update

Java 10 introduced var for local variable type inference. 

  // Java 10+ feature
  var message = "Hello Java"; // compiler infers String
  var number = 42;            // compiler infers int
⚠️ Note: var can only be used for local variables inside methods, not for class/instance/static fields.

Enhancement Tips

- Use meaningful variable names (follow Java naming conventions).
- Prefer constants (final static) for fixed values like PI, URLs, configuration keys.
- Avoid unnecessary static variables in multi-threaded apps (risk of shared-state issues).
- Leverage var for cleaner code, but don’t overuse it (can reduce readability).
Note- Java 17–21 Updates: Pattern Matching for switch (Java 21) can simplify type checks using variables directly.
