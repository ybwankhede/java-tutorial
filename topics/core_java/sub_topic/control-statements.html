Control Statements

Control statements in Java determine the flow of program execution. They help make decisions, repeat tasks, or skip certain code blocks based on conditions.

Decision-Making Statements (if, else if, and Nested if)

Definition:
The if statement in Java is a decision-making control statement. It allows your program to execute certain parts of the code only when a condition is true.

        int number = 5;
        if (number % 2 == 0) {
            System.out.println("Even Number");
        } else {
            System.out.println("Odd Number");
        }

3. if-else if-else Example

        // Example: if-else if
        int marks = 82;
        if (marks >= 90) {
            System.out.println("Grade A");
        } else if (marks >= 75) {
            System.out.println("Grade B");
        } else if (marks >= 50) {
            System.out.println("Grade C");
        } else {
            System.out.println("Fail");
        }

4. Nested if Example

        // Example: Nested if
        boolean hasID = true;
        int age = 19;

        if (hasID) {
            if (age >= 18) {
                System.out.println("Entry allowed.");
            } else {
                System.out.println("You are underage.");
            }
        } else {
            System.out.println("No ID - Entry denied.");
        }

Advantages

  • Simple to understand and implement.
  • Works well for small decision-making logic.
  • Directly supports complex decisions using nesting.

Disadvantages

  • Too many if-else blocks make code hard to read.
  • Nested if statements increase complexity.
  • Better alternatives exist like switch (for multiple cases) or polymorphism.

specific Notes

  • if works with boolean expressions only in Java (unlike C/C++ where non-zero = true).
  • Curly braces { } are optional if there is only one statement.
  • if-else chain is often replaced by switch when multiple discrete values are checked.

Real-Life Example

        // Example: ATM Withdrawal
        double balance = 1000.0;
        double withdraw = 500.0;

        if (withdraw <= balance) {
            balance -= withdraw;
            System.out.println("Transaction Successful. Remaining balance: " + balance);
        } else {
            System.out.println("Insufficient Funds.");
        }

Latest Java Enhancements

  • Java 14+ introduced switch expressions, which are more powerful and often replace long if-else chains.
  • Pattern Matching for instanceof (Java 16+) reduces the need for if type checks with casting.

Enhancement Tips

- Use early return instead of deeply nested if to improve readability.
- Refactor multiple if-else into switch expressions or polymorphism.
- Always write conditions clearly (if (salary >= 5000) instead of cryptic checks).

Switch Statement

Definition:
The switch statement in Java is a multi-branch decision-making statement. It allows you to execute one block of code among many options based on the value of a variable or expression.

Simple Understanding

  • Button 1 â†’ Switches to Sports channel

Suppose you are building a coffee machine. Options:

  • 1 â†’ Espresso
  • 2 â†’ Cappuccino
  • 3 â†’ Latte

Basic Syntax

                break;
            case value2:
                // code block
                break;
            default:
                // code if none of the cases match
        }

                }
        
                System.out.println("Day: " + dayName);
            }
        }

Latest Java Enhancements

                System.out.println("Days in month: " + days);

        }

Advantages

  • Cleaner than multiple if-else-if statements.
  • Easier to read and maintain for many conditions.
  • Supports multiple values in Java 12+ switch expressions.
  • Can return values directly (Java 14+).

Disadvantages

  • Older switch (before Java 12) only worked with primitive types, enums, and strings.
  • May cause fall-through bugs if break is forgotten in traditional switch.
  • Still less flexible than if-else for complex boolean conditions.

Specific Keywords & Concepts

  • switch â†’ main keyword
  • case â†’ different options
  • break â†’ stops execution
  • default â†’ fallback option
  • yield â†’ returns a value in switch expression (Java 13+)

Enhancement Tips

- Prefer switch expressions in Java 14+ for cleaner code.
- Use enum with switch for type safety.
- Avoid fall-through logic unless explicitly intended.
- Combine multiple cases with , (Java 12+).
- Use switch for state machines, command routing, and menu-driven programs.

Looping Statements

Definition:
Looping statements in Java are used when we want to execute a block of code repeatedly until a certain condition is met. Instead of writing the same code multiple times, loops help us save time, reduce redundancy, and make programs cleaner.

Simple Understanding

Imagine you have to eat 5 apples, one after another. Instead of saying "eat apple" 5 times, you can say: "while I still have apples, eat one." Thatâ€™s exactly what loops do â€“ repeat a task until the condition is false.

2. while loop

            public static void main(String[] args) {

                    System.out.println("Banana " + count);

                    System.out.println("Orange " + count);

                for (String fruit : fruits) {
                    System.out.println("Fruit: " + fruit);
                }
            }
        }

Disadvantages

  • Incorrect conditions may lead to infinite loops.
  • Harder to debug if loop logic is complex.
  • May cause performance issues if not optimized.

  • Streams API (Java 8+) can replace traditional loops with functional style.

Enhancement Tips

- Use Streams API (map, filter, forEach) for cleaner and parallel execution.
