Data Types — Primitive & Non-Primitive

Definition:
In Java, data types tell the compiler what kind of data a variable can hold. They define the size, kind of values, and operations that can be performed on data.

Simple Understanding

Think of data types as containers of different shapes. - A byte is like a small cup that can hold small numbers. - An int is like a bucket that can hold bigger numbers. - A String is like a notebook that stores text.

Categories of Data Types in Java

  • Primitive Data Types → Built-in, simple values (numbers, characters, true/false).
  • Non-Primitive (Reference) Data Types → Objects and more complex data (Strings, Arrays, Classes).

Primitive Data Types

Type Size Default Value Range Example
byte 8-bit 0 -128 to 127 byte b = 100;
short 16-bit 0 -32,768 to 32,767 short s = 20000;
int 32-bit 0 -2^31 to 2^31-1 int n = 500000;
long 64-bit 0L -2^63 to 2^63-1 long big = 123456789L;
float 32-bit 0.0f ≈ 7 decimal digits float pi = 3.14f;
double 64-bit 0.0d ≈ 15 decimal digits double d = 2.718281828;
char 16-bit '\u0000' Unicode 0 to 65535 char grade = 'A';
boolean 1-bit (JVM dependent) false true / false boolean isJavaFun = true;

Non-Primitive Data Types

  • String: String name = "Yogesh";
  • Array: int[] nums = {1,2,3};
  • Class: class Car { }
  • Interface, Enum, Record, etc.

Real-Life Example

Imagine you are creating a Bank Application: - Account Number → long - Balance → double - Customer Name → String - Account Active → boolean

Advantages

  • Improves performance (primitive types are fast).
  • Strong type checking prevents errors.
  • Memory-efficient (size is fixed per type).

Disadvantages

  • Primitive types can’t store complex objects.
  • Wrapper classes needed for OOP features (autoboxing/unboxing).

Specific Notes

  • All data types in Java are statistically typed (decided at compile time).
  • Wrapper Classes exist for all primitives (e.g., Integer, Double).
  • Autoboxing and Unboxing automatically convert between primitives and wrappers.
  • Since Java 10, you can use var for type inference, but actual type is still decided by compiler.

Enhancement Tips

- Prefer var (Java 10+) for local variables to improve readability.
- Use BigDecimal for precise financial calculations (avoids floating-point errors).
- Understand memory usage: choose int vs long wisely for scalability.
- Be careful when mixing primitive and wrapper types to avoid NullPointerException.
