Access Specifiers

Definition:
Access Specifiers (also called Access Modifiers) in Java define the visibility or scope of classes, methods, variables, and constructors. In simple words, they control "who can use what" in your program.

Simple Understanding

Imagine you own a house:

  • public → Anyone can enter (friends, delivery, guests).
  • protected → Only your family members can access.
  • default (no keyword) → Only neighbors inside your society can come.
  • private → Only you can enter (restricted to yourself).
Similarly, Java uses access specifiers to protect and organize code.

Real-Life Example

In a Bank Application:

  • private → Account PIN (only user knows).
  • default → Branch address (shared within the system).
  • protected → Employee salary (visible in HR module & subclasses).
  • public → Bank Name & IFSC code (anyone can see).

Types of Access Specifiers

Specifier Within Class Within Package Subclass (Different Package) Other Packages
public ✔️ ✔️ ✔️ ✔️
protected ✔️ ✔️ ✔️ ❌
default (no keyword) ✔️ ✔️ ❌ ❌
private ✔️ ❌ ❌ ❌

Example

    class Person {
        private String name = "John";     // Only inside class
        String city = "Mumbai";           // Default - same package
        protected int age = 30;           // Package + Subclass
        public String country = "India";  // Everywhere
    
        private void showName() {
            System.out.println("Name: " + name);
        }
    
        public void showCountry() {
            System.out.println("Country: " + country);
        }
    }
    
    public class AccessDemo {
        public static void main(String[] args) {
            Person p = new Person();
            // System.out.println(p.name);   // ❌ Error: private
            System.out.println(p.city);      // ✔️ Default - same package
            System.out.println(p.age);       // ✔️ Protected - same package
            System.out.println(p.country);   // ✔️ Public - everywhere
        }
    }

Advantages

  • Provides data security and controlled access.
  • Supports Encapsulation principle in OOP.
  • Makes code more organized and reusable.

Disadvantages

  • Too much restriction can make code difficult to test or extend.
  • Confusion for beginners about which modifier to use.

Specific Keywords

  • public
  • private
  • protected
  • default (when no keyword is used)

Latest Java Updates

Access modifiers remain the same in latest Java versions (Java 8 → 21). However, they are widely used with new features like:

  • Records → Members are implicitly private final.
  • Sealed Classes → Work with protected and public to control inheritance.
  • Modules (Java 9+) → Provide strong encapsulation beyond traditional specifiers.

Enhancement Tips

- Use private with getter/setter methods to follow Encapsulation.
- Restrict visibility as much as possible → "least privilege principle".
- In large projects, combine modules + access specifiers for better security.
- Don’t make everything public, it breaks modular design.
