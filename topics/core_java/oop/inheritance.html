Inheritance

Definition:
Inheritance is one of the four main principles of Object-Oriented Programming (OOP). It allows a new class (child/subclass) to reuse the fields and methods of an existing class (parent/superclass). In Java, inheritance is achieved using the extends keyword.

Simple Understanding

Think of inheritance like a family tree: a child inherits qualities (like eye color, height) from their parents. Similarly, in Java, a subclass inherits variables and methods from its parent class.

Example: A Car is a type of Vehicle. A Dog is a type of Animal. This is called an “is-a” relationship.

Real-Life Example

Let’s say we have a BankAccount class. A SavingsAccount and CurrentAccount both share common features like accountNumber and balance. Instead of writing the same code twice, we put shared code in BankAccount and let SavingsAccount and CurrentAccount inherit it.

Example

      // Parent class
      class Vehicle {
          String brand = "Generic Vehicle";
          void drive() {
              System.out.println("This vehicle is moving...");
          }
      }
  
      // Child class
      class Car extends Vehicle {
          String model = "Sedan";
          void honk() {
              System.out.println("Beep! Beep!");
          }
      }
  
      public class InheritanceExample {
          public static void main(String[] args) {
              Car myCar = new Car();
              System.out.println("Brand: " + myCar.brand);   // Inherited from Vehicle
              myCar.drive();   // Inherited method
              myCar.honk();    // Child's own method
          }
      }

Advantages

  • Code reusability – write once, reuse in child classes.
  • Logical hierarchy – organizes real-world relationships.
  • Extensibility – add new features easily by extending classes.
  • Polymorphism – subclass objects can be treated as parent type.

Disadvantages

  • Tight coupling between parent and child classes.
  • If parent class changes, child classes may break (fragile design).
  • Can be misused leading to deep inheritance trees → hard to maintain.
  • Multiple inheritance not supported with classes (only via interfaces).

Focused Keywords

  • extends → used for class inheritance
  • super → refers to parent class methods/constructors
  • this → refers to the current object
  • final → prevents inheritance
  • abstract → base class with unimplemented methods
  • Superclass (Parent Class): The class whose features are inherited.
  • Subclass (Child Class): The class that inherits features from the superclass.
  • Keyword: extends is used for class inheritance.
  • Single Inheritance: A class inherits from one superclass.
  • Multilevel Inheritance: A class inherits from another class, which itself inherits from another.
  • Hierarchical Inheritance: Multiple classes inherit from one superclass.
  • Java Limitation: Java does not support multiple inheritance with classes (only via interfaces) to avoid ambiguity (Diamond Problem).

Updated Java Features

  • Records (Java 16+): final classes that inherit from java.lang.Record, used for data carrier objects.
  • Sealed Classes (Java 17+): restrict which classes can extend a parent class using permits.
  • Java 17+ - Pattern matching in instanceof improves type checks in inherited structures.
  • Multiple Inheritance with Interfaces: Java allows multiple interfaces to be implemented, solving the "diamond problem".

Enhancement Tips

  • Prefer Composition over Inheritance: Use inheritance only when there is a clear “is-a” relationship. For flexible design, composition is often better.
  • Keep Inheritance Shallow: Avoid deep inheritance hierarchies (more than 2–3 levels), as they make debugging and maintenance difficult.
  • Use Abstract Classes for Templates: If you want partial implementation with some abstract methods, choose abstract classes.
  • Use Interfaces for Multiple Behaviors: Instead of forcing everything into a parent class, use interfaces to add extra capabilities.
  • Sealed Classes (Java 17+): If you want controlled inheritance, sealed classes are a good modern choice.
  • Avoid Inheriting Just for Reuse: Don’t extend a class only to reuse methods/fields; extract common code into utility/helper classes instead.
  • Follow Liskov Substitution Principle (SOLID): Subclasses should be usable wherever the parent is expected without breaking functionality.
