Deque (Double-Ended Queue)

Definition:
A Deque (pronounced "deck") is a double-ended queue in Java that allows insertion and deletion of elements from both ends (front and rear). It supports both FIFO (like Queue) and LIFO (like Stack).

Simple Understanding

Think of Deque as a flexible line. People can enter or leave from both the front and back of the line. Unlike Queue (one way) or Stack (single top), Deque gives two-way access.

Real-life Example

- Bus with two doors: Passengers can board or exit from the front door or the back door.
- Double-ended ticket counter: You can serve customers from either end of the counter.

Example

  import java.util.*;
  
  public class DequeExample {
      public static void main(String[] args) {
          Deque deque = new ArrayDeque<>();
  
          // Insert at both ends
          deque.addFirst("Front1");
          deque.addLast("Rear1");
          deque.offerFirst("Front2");
          deque.offerLast("Rear2");
  
          System.out.println("Deque: " + deque);
  
          // Remove from both ends
          System.out.println("Removed Front: " + deque.pollFirst());
          System.out.println("Removed Rear: " + deque.pollLast());
          System.out.println("Deque after removal: " + deque);
      }
  }

Advantages & Disadvantages

  • Advantages:
    • Supports both Queue (FIFO) and Stack (LIFO) behavior.
    • ArrayDeque is faster than Stack & LinkedList in most cases.
  • Disadvantages:
    • No random access like List (you can’t directly fetch by index).
    • ArrayDeque does not allow null elements.

Specific Keywords & Concepts

  • addFirst()/offerFirst(): Insert at the front.
  • addLast()/offerLast(): Insert at the rear.
  • pollFirst()/removeFirst(): Remove from the front.
  • pollLast()/removeLast(): Remove from the rear.
  • peekFirst()/peekLast(): View elements from both ends.

Latest Java Updates

  • Java 8: Added forEach(), streams support for Deque.
  • Java 9+: Factory methods for List/Set/Map, but no direct Deque.of().
  • Java 21: Performance improvements for concurrent deques.

Notes

- ArrayDeque is a good replacement for Stack.
- LinkedList also implements Deque but is slower compared to ArrayDeque.
- Thread-safe versions: Use ConcurrentLinkedDeque.

Tips

- Use ArrayDeque when you need a faster stack/queue hybrid.
- Avoid using LinkedList unless frequent insertions/removals are required in the middle.
- For concurrency, use BlockingDeque (like LinkedBlockingDeque).

Realistic Example for Beginners

  // Example: Browser history navigation using Deque
  Deque history = new ArrayDeque<>();
  
  // Visiting pages
  history.addLast("Google");
  history.addLast("YouTube");
  history.addLast("ChatGPT");
  
  System.out.println("Current Page: " + history.peekLast());
  
  // Back navigation
  System.out.println("Going back: " + history.pollLast());
  System.out.println("Now at: " + history.peekLast());
