Unchecked Exceptions

Definition:
Unchecked Exceptions are exceptions that occur at runtime. They are not checked by the compiler at compile time. These usually represent programming errors like dividing by zero, accessing an array out of bounds, or null pointer access.

All unchecked exceptions are subclasses of RuntimeException class.

Easy Understanding (Real-Life Analogy)

Imagine you are driving a car. You forget to wear your seatbelt. The car will not stop you from driving (compiler doesn‚Äôt check it). But if an accident happens (runtime issue), you face the consequences. This is exactly what unchecked exceptions are ‚Äî they happen only during execution.

Common Types of Unchecked Exceptions

  • NullPointerException ‚Äì when you try to use an object reference that is null.
  • ArithmeticException ‚Äì like dividing a number by zero.
  • ArrayIndexOutOfBoundsException ‚Äì when accessing invalid array index.
  • ClassCastException ‚Äì when wrong casting is done between incompatible classes.
  • IllegalArgumentException ‚Äì when passing illegal or inappropriate arguments to a method.

Example

public class UncheckedExample {
    public static void main(String[] args) {
        // NullPointerException
        String text = null;
        try {
            System.out.println(text.length()); 
        } catch (NullPointerException e) {
            System.out.println("Caught: " + e);
        }

        // ArithmeticException
        try {
            int result = 10 / 0;
        } catch (ArithmeticException e) {
            System.out.println("Caught: " + e);
        }

        // ArrayIndexOutOfBoundsException
        try {
            int[] arr = {1, 2, 3};
            System.out.println(arr[5]);
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("Caught: " + e);
        }
    }
}
