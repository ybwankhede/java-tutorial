Exception Handling Keywords

Exception handling in Java is mainly done using try, catch, finally, throw, and throws keywords. These provide a structured way to handle runtime errors, so the application doesnâ€™t crash unexpectedly.

1. try-catch Block

Definition: The try block is where you place risky code. The catch block is used to handle exceptions if they occur.

Simple Analogy: Imagine youâ€™re crossing the road (try). If an accident happens, an ambulance (catch) comes to handle the situation.

// Example: try-catch
public class TryCatchExample {
    public static void main(String[] args) {
        try {
            int result = 10 / 0; // risky code
        } catch (ArithmeticException e) {
            System.out.println("Oops! Division by zero is not allowed.");
        }
    }
}

Try Block --> Catch Block
  [Risky code]   [Handle Exception]
If no exception occurs, the catch block is skipped.

2. Nested try-catch

Definition: A try block inside another try block. Useful when multiple risky operations exist in a single method.

Real-life Example: While cooking (outer try), youâ€™re using both a knife (inner try-1) and stove (inner try-2). Both have different risks and handling methods.

public class NestedTry {
    public static void main(String[] args) {
        try {
            try {
                int arr[] = {1,2,3};
                System.out.println(arr[5]); // inner exception
            } catch(ArrayIndexOutOfBoundsException e) {
                System.out.println("Array index issue handled.");
            }
            int result = 10 / 0; // outer exception
        } catch(ArithmeticException e) {
            System.out.println("Arithmetic issue handled.");
        }
    }
}
Nested try helps in localizing error handling.

3. Multiple catch Blocks

Definition: One try can have multiple catch blocks to handle different exception types separately.

try {
   String str = null;
   System.out.println(str.length()); // NullPointerException
} catch(ArithmeticException e) {
   System.out.println("Arithmetic error.");
} catch(NullPointerException e) {
   System.out.println("Null pointer error.");
}
While multiple catch block follow exception class hierarchy.

4. finally Block

Definition: The finally block is always executed, whether an exception occurs or not. Used for resource cleanup.

Analogy: No matter if your train reaches on time or is delayed (try/catch), you will leave the station eventually (finally).

try {
    int result = 10/0;
} catch(Exception e) {
    System.out.println("Handled: " + e);
} finally {
    System.out.println("This will always run (cleanup).");
}
finally is mostly used to close files, DB connections, sockets.

5. throw Keyword

Definition: throw is used to explicitly throw an exception.

public class ThrowExample {
    static void checkAge(int age) {
        if(age < 18) {
            throw new ArithmeticException("Not eligible for voting");
        }
        System.out.println("Eligible for voting");
    }

    public static void main(String[] args) {
        checkAge(16);
    }
}
Used for custom conditions in business logic.

6. throws Keyword

Definition: Declares exceptions in method signature so caller knows it might throw exception.

public void readFile() throws IOException {
    FileReader fr = new FileReader("abc.txt");
}
throws tells the caller to handle it or propagate it further.

Q: Difference between throw and throws? A: throw is used inside method to create/throw exception. throws is used in method signature to declare exception.

Enhancement Tips

Use specific exceptions, not just generic Exception.
Always close resources using try-with-resources instead of finally.
Create custom exceptions for business logic errors.
Avoid overusing exceptions for control flow.
