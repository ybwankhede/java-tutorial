Checked Exceptions

Definition:
A Checked Exception in Java is an exception that the compiler forces you to handle (either with try-catch or by declaring throws in the method signature).

Simple Understanding

Imagine you are booking a train ticket online. If the payment gateway is down, your program must handle this situation (otherwise your booking fails silently). Similarly, in Java, checked exceptions make sure that you prepare for possible problems.

Examples of Checked Exceptions

  • IOException ‚Üí When file/network operations fail
  • SQLException ‚Üí When database access fails
  • ClassNotFoundException ‚Üí When a class is not found at runtime
  • InterruptedException ‚Üí When a thread is interrupted

Example

import java.io.*;

public class CheckedExample {
    public static void main(String[] args) {
        try {
            FileReader file = new FileReader("data.txt");  // may throw FileNotFoundException
            BufferedReader br = new BufferedReader(file);
            System.out.println(br.readLine());
        } catch (IOException e) {  // must handle IOException
            System.out.println("Error reading file: " + e.getMessage());
        }
    }
}

üéØ Interview Questions

Q1: What is a checked exception? A: An exception that must be handled at compile-time using try-catch or throws.

Q2: Can you name 3 checked exceptions in Java? A: IOException, SQLException, ClassNotFoundException.

Q3: Difference between checked and unchecked exceptions? A: Checked exceptions are checked at compile-time, unchecked at runtime.

Q4: Why did Java designers create checked exceptions? A: To make programs more reliable by forcing developers to handle failures.

Q5 (advanced): How do checked exceptions affect API design? A: They make API contracts explicit but may reduce developer convenience.
