Exceptions

Definition:
An Exception in Java is an unexpected event that disrupts the normal flow of a program. It usually happens when something goes wrong at runtime (like dividing by zero, file not found, or invalid input).

👉 Think of an Exception like a traffic signal: if you break the rule, the system (Java) stops you and raises a warning.

Simple Understanding

Imagine you are withdrawing money from an ATM:

  • If you enter the correct PIN → transaction succeeds.
  • If you enter a wrong PIN → machine throws an error message (Exception).
Similarly, Java throws an Exception when the program faces an invalid situation.

Exception Hierarchy (UML View)

  
    Throwable  
    ├── Error (Serious issues, usually unrecoverable)  
    │   ├── OutOfMemoryError  
    │   ├── StackOverflowError  
    │   ├── VirtualMachineError  
    │   └── LinkageError  
    │  
    └── Exception (Problems the program might want to catch)  
        ├── Checked Exceptions (Must be handled or declared)  
        │   ├── IOException  
        │   ├── SQLException  
        │   └── ClassNotFoundException  
        │  
        └── Unchecked Exceptions (Runtime exceptions)  
            ├── NullPointerException  
            ├── ArithmeticException  
            ├── ArrayIndexOutOfBoundsException  
            └── IllegalArgumentException

Types of Exceptions

  • Checked Exceptions → Checked at compile time (e.g., FileNotFoundException).
  • Unchecked Exceptions → Occur at runtime (e.g., NullPointerException, ArrayIndexOutOfBoundsException).
  • Errors → Serious issues, not recoverable (e.g., OutOfMemoryError).
📝 Checked = Compile-time, must handle.
Unchecked = Runtime, may/may not handle.

Java Exception Handling Keywords

  • try → Block of code that may throw exception.
  • catch → Handles the exception.
  • finally → Always executes (closing resources).
  • throw → Used to explicitly throw an exception.
  • throws → Declares exceptions in method signature.

Example Code

public class ExceptionDemo {
    public static void main(String[] args) {
        try {
            int result = 10 / 0; // risky code
        } catch (ArithmeticException e) {
            System.out.println("Cannot divide by zero: " + e.getMessage());
        } finally {
            System.out.println("Finally block always runs (e.g., closing ATM card).");
        }
    }
}

Realistic Example

Suppose you are reading a file that does not exist: 

import java.io.*;

public class FileReadDemo {
    public static void main(String[] args) {
        try {
            FileReader fr = new FileReader("myfile.txt");
        } catch (FileNotFoundException e) {
            System.out.println("File not found! Please check the path.");
        }
    }
}

✅ Advantages

  • Helps to handle runtime errors smoothly.
  • Keeps program running instead of crashing.
  • Makes debugging easier.

❌ Disadvantages

  • Overusing try-catch can make code messy.
  • Poorly handled exceptions may hide real issues.

Latest Java Updates

  • Java 7: Multi-catch block (catch(IOException | SQLException e)).
  • Java 7: Try-with-resources for auto closing files, DB connections.
  • Java 14+: Helpful NullPointerException messages (shows variable name).

Enhancement Tips

✅ Always use custom exceptions for business rules.
✅ Use logging frameworks (SLF4J, Log4j) instead of System.out.println.
✅ Keep catch blocks specific instead of using generic Exception.
✅ Use try-with-resources for database and file handling.
