Exceptions

Definition:
An Exception in Java is an unexpected event that disrupts the normal flow of a program. It usually happens when something goes wrong at runtime (like dividing by zero, file not found, or invalid input).

Types of Exceptions

  • Checked Exceptions â†’ Checked at compile time (e.g., FileNotFoundException).
  • Unchecked Exceptions â†’ Occur at runtime (e.g., NullPointerException, ArrayIndexOutOfBoundsException).
  • Errors â†’ Serious issues, not recoverable (e.g., OutOfMemoryError).

Java Exception Handling Keywords

  • try â†’ Block of code that may throw exception.
  • catch â†’ Handles the exception.
  • finally â†’ Always executes (closing resources).
  • throw â†’ Used to explicitly throw an exception.

Example Code

public class ExceptionDemo {
    public static void main(String[] args) {
        try {
            int result = 10 / 0; // risky code
        } catch (ArithmeticException e) {
            System.out.println("Cannot divide by zero: " + e.getMessage());
        } finally {
            System.out.println("Finally block always runs (e.g., closing ATM card).");
        }
    }
}
